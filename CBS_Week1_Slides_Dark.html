From generative to agentic

A new class of AI: systems that don’t just generate, they act

Generative AI

You ask → it answers

Single turn. Predicts the next token. Stateless. Cannot take action in the world.

Agentic AI

You ask → it plans → it acts → it adjusts

Multi-step. Uses tools (search, APIs, files, browsers). Maintains state. Can take real actions on real systems.

The same underlying LLM — wrapped in a planning loop with tool access

How an agentic system works

Agentic AI — benefits

End-to-end automation

Multi-step tasks complete without human intervention at every stage

Adaptive to context

Can handle ambiguous goals and unexpected obstacles by replanning

Tool integration

Uses existing software (APIs, databases, browsers) without custom integration code

Faster knowledge work

Research, analysis, drafting, and routine decision-making at machine speed

Agentic AI — business use cases

Customer service resolution

Agent reads ticket, checks order DB, issues refund, sends email — one workflow

Financial research & reporting

Pulls earnings data, summarises filings, generates draft analyst report

Coding & development

Reads codebase, identifies bug, writes fix, runs tests, opens pull request

Compliance & due diligence

Reviews contracts, flags risks, cross-checks against regulations, drafts memos

Agentic AI — pitfalls

Compounding errors

A 95%-accurate step run 10 times is 60% accurate end-to-end. Long workflows fail silently.

Real-world side effects

Agents that act on real systems can issue refunds, send emails, or delete files in error

Prompt injection & tool abuse

Malicious instructions in fetched content can hijack the agent’s tools

Accountability gap

When an autonomous agent makes a bad decision, who is responsible? Legal frameworks are unclear.

Rule of thumb: human-in-the-loop wherever the action is irreversible, costly, or regulated
